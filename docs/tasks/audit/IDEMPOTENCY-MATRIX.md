## Key Findings

- Strong database-backed idempotency exists for user, settings, referral, plan selection, payments, manual instruction, receipt upload, and outbox creation.
- Outbox processing now uses an atomic PostgreSQL claim through `ProvisioningOutboxRepository#claimAvailableByEventId`, backed by `FOR UPDATE SKIP LOCKED` and covered by `ConcurrentProvisioningOutboxClaimIntegrationTest`.
- XUI operation idempotency has a durable `operation_id`, fingerprint, locked provision reads, and deterministic PostgreSQL concurrency coverage.
- Payment approval relies on a correct partial unique index for the critical invariant: only one approved payment per order.
- Final verification reran focused concurrency tests on 2026-07-13; all passed.
