CreateSubscriptionUseCase

Because raw tokens are not stored, repeated creation cannot reproduce the original token.

* first creation:

  * `newlyCreated=true`

* replay:

  * `newlyCreated=false`
  * rawToken absent;
  * client may explicitly rotate the token if a new raw token is needed.

Document this clearly.

# 34. Expiration reconciliation

On public access:

* If Subscription expiry is reached:

  * transition ACTIVE/SUSPENDED -> EXPIRED in a short transaction;
  * return unavailable.
* If underlying provision is:

  * DISABLED;
  * DELETING;
  * DELETED;
  * FAILED;
  * INVALID;
    then Subscription must not return usable config.

Recommended mapping:

Provision ACTIVE -> Subscription may be ACTIVE

Provision DISABLED -> Subscription SUSPENDED or unavailable

Provision DELETED -> Subscription INVALID or REVOKED according to policy

Provision UNKNOWN -> temporary unavailable

Do not mutate provision lifecycle from subscription access.

Document lifecycle reconciliation.
