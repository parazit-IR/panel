6. User aggregate design

Create the User aggregate in the domain layer.

Suggested package:

com.parazit.panel.domain.user

Create:

User

The User entity must extend BaseEntity.

Required fields:

UUID id inherited from BaseEntity

Long telegramUserId

String username

String firstName

String lastName

UserLanguage language

UserStatus status

Boolean blocked

Instant lastInteractionAt

Instant createdAt inherited from BaseEntity

Instant updatedAt inherited from BaseEntity

Design rules:

* telegramUserId is the external identity from Telegram.
* telegramUserId must be unique and required.
* username is optional because Telegram users may not have a username.
* firstName is required.
* lastName is optional.
* language is required.
* status is required.
* blocked must not be null.
* lastInteractionAt is optional initially.
* Strings must be normalized.
* Empty optional strings should be stored as null where practical.
* Do not store Telegram access tokens or secrets on the user.
* Do not add password fields.
* Do not add email or phone unless already required by the current project.
* Do not add payment or subscription fields directly to User.
* Do not add bidirectional relationships.
* Keep the aggregate small.
