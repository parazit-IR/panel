3. PlanStatus

Create:

PlanStatus

Values:

DRAFT
ACTIVE
INACTIVE
ARCHIVED

Persist it using EnumType.STRING.

Semantics:

DRAFT:
The plan is incomplete or not yet published.

ACTIVE:
The plan is available for users.

INACTIVE:
The plan is temporarily unavailable.

ARCHIVED:
The plan is permanently retired but remains in the database.

Use status as the source of truth for availability.

Do not add a separate active boolean.

Recommended transitions:

DRAFT -> ACTIVE
DRAFT -> ARCHIVED
ACTIVE -> INACTIVE
ACTIVE -> ARCHIVED
INACTIVE -> ACTIVE
INACTIVE -> ARCHIVED
ARCHIVED -> no outgoing transition

Create explicit domain methods:

activate()

deactivate()

archive()

Do not create a generic status setter.

Define and test same-state behavior clearly.

7. Traffic representation

Use:

Long trafficLimitBytes

Store traffic in bytes.

Use the binary convention:

1 GiB = 1024 × 1024 × 1024 bytes

Example:

30 GiB = 30L * 1024 * 1024 * 1024

Rules:

* Do not store GB as a floating-point value.
* Limited plans require a positive byte count.
* Unlimited plans require null.
* Conversion for display can be added in future application or API layers.
