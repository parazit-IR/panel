6. Dependency injection rules

Establish the following project-wide conventions:

* Constructor injection only
* Final dependencies
* No @Autowired on constructors when there is only one constructor
* No field injection
* No setter injection except for framework-required cases
* Domain classes must not depend on Spring
* Application services may depend on domain interfaces
* Infrastructure adapters may implement domain or application ports
* API controllers may depend on application services
* Configuration classes may create infrastructure beans
* Avoid injecting concrete infrastructure implementations into application services
* Avoid static mutable state
* Avoid global singleton service locators

Document these rules.
