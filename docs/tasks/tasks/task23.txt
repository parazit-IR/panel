28. HTTP behavior

GET list:

* 200 with array.
* Empty remote list → 200 with [].

GET by ID:

* 200 when found.
* 404 when absent.
* invalid ID → 400.

* 200 when exactly one match.
* 404 when none.
* 409 when duplicate remarks make lookup ambiguous.

GET eligible:

* 200 with one deterministic eligible inbound.
* 404 when none eligible.

Remote failures:

* authentication failure → mapped using existing Xui authentication policy.
* connection failure → 503.
* timeout → 504 if supported.
* invalid remote response → 502.
* remote 5xx → 502 or 503 according to current conventions.

All error responses must contain traceId and no secrets.
