# 3. Scheduling architecture

Create a clear separation between:

* trigger;
* lock;
* execution;
* item claiming;
* item processing;
* completion;
* observability.

Recommended structure:

`@Scheduled` trigger
-> JobCoordinator
-> distributed job lock
-> application job executor
-> bounded batch
-> per-item short transactions
-> result summary
-> job-run completion

Rules:

* Scheduler classes contain no business decision logic.
* Job executor may call use cases.
* Each job has a stable unique key.
* Every job execution is bounded.
* Every job supports disabled mode.
* Every job logs start and completion.
* Failure of one item must not necessarily fail the whole batch.
* One application instance may run the same job at a time.
* Multiple different jobs may run concurrently if safe.
* No global lock covering all jobs.
