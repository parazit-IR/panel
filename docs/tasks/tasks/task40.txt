* all previous Tasks remain functional;
* all tests pass;
* no tests are disabled;
* Flyway works from an empty database;
* Hibernate validation passes;
* production profile starts with valid configuration;
* invalid production configuration fails safely;
* admin APIs require authentication;
* roles are enforced;
* temporary operator-header authentication is not active in production;
* receipt content is role-restricted;
* one approved Payment per Order remains enforced;
* provisioning remains outbox-driven;
* no duplicate VPN client is created;
* raw Subscription tokens are not stored;
* Telegram duplicate updates remain safe;
* notifications remain idempotent;
* jobs remain multi-instance safe;
* Docker image runs non-root;
* PostgreSQL is not public in production Compose;
* secrets are external;
* Nginx/HTTPS configuration is documented;
* backup scripts succeed;
* restore test succeeds;
* receipt backup is included;
* health/metrics/logging are operational;
* smoke test succeeds;
* known limitations are documented;
* no production secrets are committed.

# 91. Known limitations section

Explicitly document remaining limitations, likely including:

* single-VPS availability;
* no automatic failover;
* no multi-region database;
* Telegram exactly-once delivery is impossible;
* local filesystem receipt storage requires backup;
* HTTP Basic admin authentication may be an interim solution;
* no frontend admin dashboard;
* no automated refunds;
* no chargebacks;
* no reseller support;
* no automatic traffic add-on;
* no Kubernetes;
* no managed secret store by default.

Do not hide limitations.
