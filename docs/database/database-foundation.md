# Database Foundation

## PostgreSQL

The application uses PostgreSQL through Spring Data JPA and Hibernate. Local
development is compatible with the PostgreSQL service in `docker/docker-compose.yml`.

```bash
docker compose -f docker/docker-compose.yml up -d postgres
docker compose -f docker/docker-compose.yml ps
```

If the Docker Compose plugin is unavailable, use the installed legacy command:

```bash
docker-compose -f docker/docker-compose.yml up -d postgres
docker-compose -f docker/docker-compose.yml ps
```

## Configuration

Datasource credentials are read from environment variables:

```bash
DATABASE_URL=jdbc:postgresql://localhost:5432/panel_db
DATABASE_USERNAME=panel
DATABASE_PASSWORD=panel
```

Connection pool settings are also environment-driven:

```bash
DB_POOL_MAX_SIZE=10
DB_POOL_MIN_IDLE=2
DB_CONNECTION_TIMEOUT_MS=30000
DB_IDLE_TIMEOUT_MS=600000
DB_MAX_LIFETIME_MS=1800000
```

Only the `local` profile provides safe development defaults. `dev` and `prod`
must receive values from the runtime environment.

## Flyway

Flyway is the only tool responsible for schema changes. Hibernate schema
generation is disabled with `spring.jpa.hibernate.ddl-auto=validate`, so the
application validates the schema instead of creating or updating it.

Migration files live in:

```text
src/main/resources/db/migration
```

Use this naming convention:

```text
V<version>__<description>.sql
```

Example next migration:

```text
V2__create_example_table.sql
```

The initial migration does not enable PostgreSQL UUID extensions because UUIDs
are generated by Hibernate in the application.

## UUID Strategy

`BaseEntity` uses `UUID` identifiers generated by Hibernate with `@UuidGenerator`.
Controllers and services must not assign entity identifiers manually.

## JPA Auditing

JPA auditing is enabled in `JpaAuditingConfiguration`.

`BaseEntity` includes:

- `id`
- `createdAt`
- `updatedAt`

`createdAt` is insert-only, and `updatedAt` changes on update. Both use
`Instant` for UTC-compatible timestamps.

## Tests

Persistence tests use Testcontainers PostgreSQL and do not rely on the local
database. Test-only entities and migrations live under `src/test`.

Run:

```bash
gradle test
gradle build
```

If using a checked-in wrapper in the future, the equivalent commands are:

```bash
./gradlew test
./gradlew build
```
